# منصة إعلان النتائج

تطبيق عربي لإعداد نتائج المسابقات وعرضها مباشرة على الهواتف والشاشات الكبيرة،
مبني باستخدام React وTypeScript وVite وTailwind CSS.

## الاستخدام

1. اختر مرحلة `متوسط` أو `ثانوي` من الصفحة الرئيسية.
2. افتح `إعداد النتائج` لتعديل أسماء الأسر والدرجات.
3. اختر `معاينة العرض` بعد مراجعة تنبيهات البيانات.
4. في شاشة النتائج، اضغط على خانات الدرجات لكشفها وحساب المجموع.

في شاشة العرض:

- المفتاح `F` لتشغيل ملء الشاشة أو الخروج منه.
- المفتاح `R` لفتح تأكيد إعادة إخفاء الدرجات.
- المفتاح `Escape` للخروج من ملء الشاشة.

## التشغيل المحلي

```bash
npm install
npm run dev
```

يمكن نسخ `.env.example` إلى `.env` لإعداد Supabase ورمز دخول صفحة الإعداد:

```dotenv
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
VITE_SETUP_PASSCODE=
```

إذا تُرك `VITE_SETUP_PASSCODE` فارغًا، تبقى صفحة إعداد النتائج متاحة دون رمز.
وإذا تُركت متغيرات Supabase فارغة، يعمل التطبيق تلقائيًا باستخدام التخزين
المحلي فقط.

## الأوامر

- `npm run dev` لتشغيل بيئة التطوير.
- `npm run build` للتحقق من الأنواع وإنشاء نسخة الإنتاج.
- `npm run lint` لفحص جودة الشفرة.
- `npm run preview` لمعاينة نسخة الإنتاج محليًا.

## البنية

- `src/components` للمكوّنات العامة ومكوّنات الواجهة القابلة لإعادة الاستخدام.
- `src/layouts` للتخطيط العام البسيط للتطبيق.
- `src/pages` لصفحات التطبيق.
- `src/store` لحالة التطبيق المركزية.
- `src/services` لطبقة الوصول إلى البيانات والتخزين.
- `src/types` لأنواع نطاق التطبيق.
- `src/constants` للثوابت ومراحل المنافسة والمسارات.
- `src/hooks` للخطافات المشتركة.
- `src/utils` للأدوات المساعدة النقية.

تستخدم الحالة مستودع Supabase عند توفر متغيرات البيئة المطلوبة، وتعود تلقائيًا
إلى التخزين المحلي عند غيابها أو تعذر الاتصال.

## النشر على Netlify

يحتوي المشروع على `netlify.toml` يحدد أمر البناء ومجلد النشر وتحويل جميع
المسارات إلى تطبيق React.

1. اربط المستودع بمشروع Netlify.
2. أضف متغيرات البيئة المطلوبة من إعدادات المشروع.
3. نفّذ مخطط قاعدة البيانات الموضح في
   [`supabase/README.md`](./supabase/README.md) عند استخدام Supabase.
4. انشر المشروع، ثم اختبر فتح `/setup` و`/results` مباشرة.

## تنبيه أمني حول رمز الإعداد

رمز `VITE_SETUP_PASSCODE` ليس حماية أمنية قوية. متغيرات بيئة Vite الأمامية
تُضمّن في ملفات المتصفح ويمكن للمستخدم الاطلاع عليها، كما يمكن تعديل
`sessionStorage` يدويًا. هذه الآلية مخصّصة فقط لمنع التعديلات العرضية أو
غير المقصودة.

للحماية الفعلية يجب لاحقًا استخدام Supabase Auth مع سياسات RLS أكثر تقييدًا.
